# 水产品店在线销售管理系统项目计划书

## 一、项目概述

本项目旨在设计并实现一个专门用于水产品行业的在线销售管理系统。系统采用 B/S 架构，使用前后端分离的开发模式。后端基于 Java 语言，结合 SSM 框架（Spring + SpringMVC + MyBatis）处理核心业务逻辑；前端使用 Vue.js 框架构建用户界面；数据层选用 MySQL 数据库进行持久化存储。

系统主要分为用户端和管理员端。用户端支持商品浏览、分类筛选、加入购物车、提交订单以及查看个人订单等功能；管理员端支持分类管理、商品管理、订单处理及用户管理等核心操作。系统通过“浏览商品 - 加入购物车 - 提交订单 - 扣减库存 - 管理员发货 - 用户确认收货”的完整业务流程，帮助水产品店提升线上运营效率，为顾客提供便捷的购物体验。

**开发工具**：IntelliJ IDEA
**项目构建**：Maven

## 三、功能模块

### 3.1 用户端功能

#### 3.1.1 首页

- 展示系统 Logo 及导航栏（首页、商品分类、购物车、登录/用户头像）。
- 从后台获取销量较高或最新上架的商品作为轮播图展示，轮播图点击后跳转到对应商品详情页。
- 显示**热门商品推荐**区域：从后台获取销量最高的 8 个商品，以卡片形式展示（图片、名称、价格、库存状态）。
- 点击卡片可跳转至商品详情页。
- 未登录用户可正常浏览首页。

#### 3.1.2 商品分类/筛选页

- 左侧显示商品分类列表（如：鱼类、虾类、贝类、蟹类、其他）。
- 支持“全部商品”与单个分类切换，点击分类后右侧展示该分类下所有商品。
- 支持按商品名称模糊搜索。
- 商品以网格列表展示，每个卡片包含：商品图片（单张）、名称、价格、库存状态。
- 点击卡片跳转到商品详情页。
- 库存为 0 的商品显示“已售罄”，仍可查看详情，但不可加入购物车。

#### 3.1.3 商品详情页

- 展示商品信息：名称、图片（一张主图）、单价、库存数量、规格/重量、商品描述。
- 提供**数量选择器**（- / 输入框 / +），最小值为 1，最大值为当前库存。
- 提供**加入购物车**按钮：将商品加入购物车（需登录，未登录则跳转登录页）。若购物车中已存在该商品，则累加数量，但总数量不能超过库存。
- 当库存为 0 时，数量选择器和加入购物车按钮置灰，提示“商品已售罄”。
- *简化说明：没有多个图片轮播，没有放大镜效果，没有“立即购买”按钮。*

#### 3.1.4 登录/注册页

- 登录：使用手机号 + 密码登录，校验通过后返回 Token 及用户信息（昵称、角色）。
- 注册：填写手机号、密码、昵称，注册成功后自动登录；手机号必须唯一，密码需进行加密存储。
- 登录成功后根据角色跳转：普通用户跳转首页，管理员跳转管理后台。
- 被禁用用户不能登录；若已登录用户被管理员禁用，后续受保护接口应拒绝访问。
- 用户头像菜单提供退出登录功能，前端清除 Token 和本地用户信息后返回首页。

#### 3.1.5 购物车页（需登录）

- 展示用户加入购物车的商品列表（商品名、单价、数量、小计）。
- 每个商品前使用多选框，支持单选、全选和取消全选。
- 支持修改商品数量或删除商品，修改数量时不能小于 1，也不能超过当前库存。
- 显示选中商品总金额，未选中商品不计入结算金额。
- 提供**结算**按钮：跳转至订单确认页，携带多选框选中的购物车项；未选择商品时按钮置灰或提示先选择商品。
- 若商品库存不足，购物车中对应商品提示异常，不能直接结算。

#### 3.1.6 订单确认页（需登录）

- 展示本次下单的商品清单（只读，不可修改）。
- 显示订单总金额。
- **收货信息**：填写收货人姓名、联系电话、详细收货地址。地址使用文本输入框，用户手动填写完整地址（如“XX省XX市XX区XX路XX号”）。*（简化：不管理多个地址，无三级联动）*
- 点击**提交订单**按钮：后端重新校验选中购物车项是否属于当前用户、商品是否存在、库存是否充足，并按当前商品价格生成订单。
- 订单创建成功后，状态为“待发货”，同时生成订单商品快照、扣减商品库存、增加商品销量，并清空购物车中对应商品。
- 若库存不足、商品不存在或地址信息为空，提交失败并给出提示，数据库操作整体回滚。
- *简化：无支付环节，提交成功即视为有效订单。*

#### 3.1.7 个人中心页（需登录）

- 顶部显示用户头像（默认占位图）、昵称、手机号（中间四位隐藏，如138****1234）。
- 左侧导航：**我的订单**（只有一个选项，无其他子菜单）。
- 右侧展示**订单列表**：每项显示订单号、下单时间、商品列表（名称、数量）、总金额、订单状态（待发货/待收货/已完成）。
- 操作按钮：
  - 待发货订单：无操作，等待管理员发货。
  - 待收货订单：可点击“确认收货”，状态变为“已完成”。
  - 已完成订单：仅展示。
- *简化：无订单状态Tab切换，无分页。*

### 3.2 管理员端功能

#### 3.2.1 后台首页（仪表盘）

- 以卡片形式展示统计数据：用户总数、商品总数、订单总数、待发货订单数。
- *简化：无定时刷新，页面加载时请求一次。*

#### 3.2.2 商品管理

- 表格展示所有商品（ID、商品名称、分类、价格、库存、规格、图片URL、销量）。
- 提供新增商品、编辑商品、删除商品操作。
- 新增/编辑弹窗字段：商品名称、所属分类（下拉选择）、价格、库存、规格、商品图片URL（直接输入链接）、商品描述。
- 商品价格必须大于 0，库存必须为不小于 0 的整数，分类必须存在。
- 删除商品后，已生成订单通过订单商品快照保留原商品信息；购物车中的对应商品项自动移除。
- 支持按商品名称模糊搜索。
- *简化：无分页、无批量操作、无分类级别限制。*

#### 3.2.3 订单管理

- 表格展示所有订单（订单号、用户昵称、手机号、商品件数、总金额、下单时间、订单状态）。
- 顶部提供按订单状态筛选的下拉框（全部/待发货/待收货/已完成）。
- 点击订单号可查看订单详情，包括收货人、联系电话、收货地址和商品明细，便于管理员发货核对。
- 操作按钮：
  - 待发货订单：显示“发货”按钮，点击后订单状态变为“待收货”。
  - 其他状态：无操作。
- *简化：无时间范围筛选、无物流单号录入。*

#### 3.2.4 用户管理

- 表格展示所有注册用户（ID、昵称、手机号、注册时间、状态（正常/禁用））。
- 提供**禁用/启用**按钮，点击切换用户状态（禁用后用户无法登录）。
- 管理员账号不允许在页面中被普通启用/禁用操作误禁用。
- *简化：无搜索、无删除、无角色修改。*

#### 3.2.5 分类管理

- 表格展示商品分类（ID、分类名称、创建时间）。
- 支持新增、编辑、删除分类。
- 删除分类前需要判断该分类下是否存在商品；若存在商品，则禁止删除并提示先调整商品分类。
- 商品管理中的分类下拉框从分类表中读取，避免前端写死分类数据。

### 3.3 核心业务逻辑与校验

- **登录鉴权**：普通用户接口需要校验 Token；管理端接口除校验 Token 外，还需要校验用户角色为管理员。
- **数据归属校验**：用户只能操作自己的购物车项和订单，不能通过传入其他用户的 ID 查看或修改他人数据。
- **订单事务**：提交订单时，创建订单、创建订单商品快照、扣减库存、增加销量、删除已结算购物车项必须放在同一个事务中，任一环节失败则全部回滚。
- **金额计算**：订单金额以后端查询到的商品当前价格为准，前端传入金额只用于页面展示，不能作为最终入库金额。
- **库存校验**：加入购物车、修改购物车数量、提交订单三个环节都需要校验库存；提交订单时以数据库实时库存为准，扣减库存时使用事务和行锁或条件更新（`stock >= quantity`）防止超卖。
- **状态流转**：订单状态只能按“待发货 -> 待收货 -> 已完成”流转，管理员只能执行发货，用户只能执行确认收货。
- **参数校验**：手机号格式、密码长度、商品价格、库存数量、购买数量、收货信息长度等均需前后端同时校验，后端校验作为最终依据。

-- 水产品店在线销售管理系统数据库初始化脚本
-- 说明：如果重复执行，本脚本会先删除旧表再重新创建，请不要在已有正式数据的数据库中直接运行。

CREATE DATABASE IF NOT EXISTS fish_shop_db DEFAULT CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;

USE fish_shop_db;

SET FOREIGN_KEY_CHECKS = 0;
DROP TABLE IF EXISTS `order_item`;

    `id` INT(11) NOT NULL AUTO_INCREMENT,

    `nickname` VARCHAR(50) NOT NULL COMMENT '用户昵称',
    `phone` VARCHAR(20) NOT NULL COMMENT '手机号',
    `password` VARCHAR(255) NOT NULL COMMENT '密码（加密存储）',

CREATE TABLE `category` (

## 五、接口设计（简要）

以下为后端 RESTful API 核心接口（部分）。除登录、注册、商品浏览和分类浏览外，其余用户接口需要携带 Token；所有 `/api/admin/**` 接口需要管理员权限。

| 模块     | 请求方式 | 路径                         | 说明                                           |
| :------- | :------- | :--------------------------- | :--------------------------------------------- |
| 认证     | POST     | /api/auth/login              | 登录，返回 Token 和用户信息                    |
|          | POST     | /api/auth/register           | 注册                                           |
| 用户     | GET      | /api/users/me                | 获取当前用户信息                               |
| 商品分类 | GET      | /api/categories              | 获取所有分类                                   |
| 商品     | GET      | /api/products                | 获取商品列表，支持 `categoryId`、`keyword` 查询 |
|          | GET      | /api/products/{id}           | 获取商品详情                                   |
| 购物车   | GET      | /api/cart/items              | 获取当前用户购物车列表                         |
|          | POST     | /api/cart/items              | 添加商品到购物车                               |
|          | PUT      | /api/cart/items/{id}         | 修改购物车商品数量                             |
|          | DELETE   | /api/cart/items/{id}         | 删除购物车项                                   |
| 订单     | POST     | /api/orders                  | 提交订单，入参包含购物车项ID、收货信息         |
|          | GET      | /api/orders/my               | 获取当前用户订单列表                           |
|          | GET      | /api/orders/{id}             | 获取当前用户订单详情                           |
|          | PUT      | /api/orders/{id}/confirm     | 用户确认收货                                   |
| 管理端   | GET      | /api/admin/dashboard         | 获取首页统计数据                               |
| 分类管理 | GET      | /api/admin/categories        | 获取分类列表                                   |
|          | POST     | /api/admin/categories        | 新增分类                                       |
|          | PUT      | /api/admin/categories/{id}   | 编辑分类                                       |
|          | DELETE   | /api/admin/categories/{id}   | 删除分类                                       |
| 商品管理 | GET      | /api/admin/products          | 获取所有商品，支持名称搜索                     |
|          | POST     | /api/admin/products          | 新增商品                                       |
|          | PUT      | /api/admin/products/{id}     | 编辑商品                                       |
|          | DELETE   | /api/admin/products/{id}     | 删除商品                                       |
| 订单管理 | GET      | /api/admin/orders            | 获取所有订单，支持状态筛选                     |
|          | GET      | /api/admin/orders/{id}       | 获取订单详情                                   |
|          | PUT      | /api/admin/orders/{id}/ship  | 管理员发货                                     |
| 用户管理 | GET      | /api/admin/users             | 获取所有用户                                   |
|          | PUT      | /api/admin/users/{id}/status | 启用/禁用用户                                  |
