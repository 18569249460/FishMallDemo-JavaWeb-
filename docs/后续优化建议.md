# 鲜渔小店后续优化建议

> 记录时间：2026-05-08  
> 目标：在现有“用户端浏览下单 + 管理端基础维护”的基础上，把项目继续打磨成更稳定、更完整、更像真实商城系统的毕业设计/课程设计项目。

## 一、优先修复项（已完成）

### 1. 前端路由与导航状态

- 已修复用户端顶部导航高亮问题：商品详情页高亮“商品分类”，订单确认页高亮“购物车”，个人中心子页面高亮“个人中心”。
- 已修复编程式跳转后的菜单状态同步问题，避免“已经进入购物车页面，但顶部仍停留在商品分类高亮”的情况。
- 管理端已独立为“顶部管理员栏 + 左侧侧边栏 + 右侧内容区”，不再复用用户端顶部导航。

### 2. 管理端页面布局

- 已按 `docs/参考.md` 的描述统一后台风格：左侧菜单、顶部管理员信息、底部版权信息、主内容卡片区。
- 后台首页已在统计数字外增加待处理订单占比图、数据概览图和快捷入口。
- 商品、分类、用户、订单页面已统一使用“页面标题 + 筛选栏 + 表格卡片 + 分页/空状态”的结构。

### 3. 启动说明与运行手册

- 已新增根目录 `README.md`，写清楚重启电脑后的启动顺序：MySQL、Tomcat 后端、Vite 前端。
- 已补充 IDEA Tomcat 文字配置步骤，避免后续每次都重新摸索。
- 已把测试账号、测试数据 SQL、访问地址集中写到 `README.md` 中。

## 二、功能完善建议（已完成一版）

### 1. 商品管理

- 后台商品列表已增加分类筛选、本地分页、库存状态标签。
- 商品图片已从“填写 URL”扩展为“填写 URL 或选择本地图片”，本地图片会上传到后端 `/uploads/images/shangpin` 目录，数据库保存图片访问地址。
- 商品已支持上下架状态，前台列表、热门商品和详情只展示上架商品。
- 商品详情已增加产地、保质期、储存方式、配送说明。

### 2. 分类管理

- 分类已增加排序字段，前台分类列表按排序展示。
- 分类删除前会显示关联商品数量，后端仍会拒绝删除有关联商品的分类。
- 分类已支持启用/禁用，前台只展示启用分类。

### 3. 购物车与订单

- 购物车已支持表格自带全选、批量删除和失效商品清理。
- 已新增订单详情页，用户可以查看收货信息、商品明细、物流单号和状态流转。
- 已增加取消订单功能，待发货订单取消后会回滚库存和销量。
- 后台发货时已可填写物流单号，用户端订单列表和详情页会展示。
- 订单状态已扩展为：待发货、待收货、已完成、已取消。

### 4. 用户中心

- 用户已可以修改昵称和头像。
- 已增加常用收货地址管理，下单时可直接选择默认地址或常用地址。
- 用户订单列表已增加状态筛选和订单号搜索。

### 5. 管理端用户管理

- 用户列表已支持按昵称、手机号搜索。
- 用户列表已增加本地分页，避免页面一次性展示过多数据。
- 用户删除采用“禁用/启用”方式替代硬删除，避免破坏历史订单关联。

## 三、后端与数据库优化

### 1. 分页与查询

- 商品、用户、订单接口建议改为分页返回，统一响应结构如：`records`、`total`、`pageNum`、`pageSize`。
- 后台订单支持按订单号、用户手机号、状态、时间范围组合查询。
- 常用查询字段增加索引，例如商品名称、分类 ID、订单号、用户 ID、订单状态。

### 2. 数据一致性

- 下单扣库存已经使用事务，后续可继续加强并发测试。
- 订单取消、删除商品、禁用用户等操作需要考虑历史订单数据是否仍可正常展示。
- 商品价格进入订单后应继续使用订单快照，避免商品改价影响历史订单。

### 3. 安全性

- 后台接口继续保持管理员权限校验，前端隐藏按钮不能作为唯一安全措施。
- 密码规则可以增强，例如最少 8 位、包含数字和字母。

### 4. 异常与日志

- 后端全局异常处理可以增加日志输出，方便排查线上问题。
- 业务异常继续返回中文提示，系统异常只返回通用提示，详细堆栈写日志。
- 关键操作建议记录日志：管理员发货、禁用用户、删除商品、删除分类。

## 四、前端体验优化

### 1. 表单体验

- 所有表单继续保持中文校验提示。
- 保存按钮防重复点击，提交中显示 loading。
- 弹窗关闭后清空校验状态，避免下次打开还残留红色错误。

### 2. 空状态与加载状态

- 空购物车、无订单、无商品、搜索无结果要区分文案。
- 空状态中提供下一步按钮，例如“去选商品”“重置筛选”“新增商品”。
- 列表加载时使用骨架屏或表格 loading，避免页面闪动。

### 3. 性能优化

- 当前 Element Plus 是全量引入，Vite 构建提示包体积较大。后续可以改为按需导入。
- 路由组件可以做懒加载，减少首页首次加载体积。
- 商品图片使用合适尺寸，避免直接加载过大的原图。

## 五、测试建议

### 1. 手动联调流程

建议每轮功能修改后都按以下流程跑一遍：

1. 普通用户注册。
2. 普通用户登录。
3. 浏览商品列表和商品详情。
4. 加入购物车。
5. 修改购物车数量。
6. 提交订单。
7. 管理员登录后台。
8. 后台查看订单并发货。
9. 普通用户确认收货。
10. 后台查看统计数据是否变化。

### 2. 后端测试

- 给认证、购物车、下单、订单状态流转补单元测试或集成测试。
- 重点测试库存不足、重复提交、越权访问、禁用用户访问等边界情况。
- 下单事务建议重点测试：任一步失败时订单、库存、购物车是否回滚。

### 3. 前端测试

- 登录拦截：未登录访问购物车、个人中心、订单确认页要跳转登录。
- 权限拦截：普通用户不能进入 `/admin`。
- 表单校验：手机号错误、密码过短、商品价格非法、库存非法都要显示中文提示。
- 空状态：无商品、无分类、无订单、无购物车项都要展示中文说明。
