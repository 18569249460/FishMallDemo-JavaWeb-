# 管理员账户添加说明

## 一、设计方式

本项目的普通注册入口只允许创建普通用户，不能通过前台注册直接生成管理员账号。管理员账号的新增放在管理后台的“账户管理”页面中，由已登录管理员操作。

这样设计更安全，也更符合真实商城后台的使用方式：后台权限由现有管理员分配，避免普通访客猜到接口或页面后自行注册管理员账号。

## 二、页面添加管理员

已有管理员登录后，可以按下面步骤新增管理员账号：

1. 使用管理员账号登录系统。
2. 登录成功后进入管理后台首页。
3. 点击左侧菜单“账户管理”。
4. 点击页面右上方“新增管理员”按钮。
5. 填写管理员昵称、手机号、登录密码、确认密码。
6. 点击“确认新增”。

新增成功后，账户列表会刷新，并显示新账号的角色为“管理员”。

页面表单会进行以下校验：

- 昵称不能为空，长度不能超过 50 个字符。
- 手机号必须是正确的中国大陆手机号格式。
- 密码长度必须为 6 到 32 位。
- 两次输入的密码必须一致。

## 三、接口说明

新增管理员接口为：

```text
POST /api/admin/users/admins
```

请求示例：

```json
{
  "nickname": "后台管理员",
  "phone": "19912345678",
  "password": "123456"
}
```

该接口位于 `/api/admin/**` 路径下，会经过管理员权限拦截器校验。只有已登录且角色为管理员的账号才能调用，普通用户调用会被拒绝。

## 四、第一位管理员怎么来

如果数据库里还没有任何管理员，就无法先登录后台新增管理员。这时需要手动执行一次初始化 SQL，创建第一位管理员。

项目已有测试脚本：

```text
docs/测试数据.sql
```

其中包含一个默认管理员：

```text
手机号：19900000000
密码：123456
```

如果只想单独补一个管理员账号，可以在 MySQL 中执行：

```sql
USE fish_shop_db;

INSERT INTO `user` (avatar, nickname, phone, password, role, status)
VALUES (
    NULL,
    '管理员',
    '19900000000',
    '123456',
    1,
    1
)
ON DUPLICATE KEY UPDATE role = 1, status = 1;
```

上面 SQL 中的 `password` 字段直接保存明文密码 `123456`。后续正式使用时，建议登录后台后新增一个自己的管理员账号，再停用或修改测试账号。
